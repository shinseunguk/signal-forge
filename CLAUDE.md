# signal-forge

LLM이 뉴스·공시를 구조화 시그널로 태깅하고, 그 시그널의 예측력을 가상 포트폴리오(초기 시드 1억원)로 검증하는 **페이퍼 트레이딩 시스템**. 실제 주문은 하지 않는다.

## 개발 기획서 (필독)

전체 시스템 스펙·아키텍처·구현 순서는 **[docs/PLAN.md](docs/PLAN.md)** 에 정의되어 있다.
코드 작업 전 반드시 이 문서를 참고하고, 각 Phase를 순서대로 구현한다.

특히 다음 절대 원칙을 위반하지 않는다 (기획서 §1):
1. 실제 주문 API(매수/매도/정정/취소)를 코드에 포함하지 않는다. 토스 API는 읽기 전용 시세 조회에만 사용.
2. LLM은 판단하지 않고 태깅만 한다. 매매 결정은 결정론적 규칙 엔진(`StrategyService`)이 내린다.
3. 모든 페이퍼 주문에 마찰비용(수수료·세금·슬리피지)을 반영한다.
4. 모든 시그널·주문은 `published_at`(원문 발행 시각)을 기록한다 (look-ahead bias 방지).

## 커밋 규칙

모든 커밋 메시지는 **Conventional Commits** 형식을 따른다.

```
<타입>[적용 범위(선택)]: <설명>

[본문(선택)]

[꼬리말(선택)] (closed #이슈번호)
```

타입: `feat` | `fix` | `refactor` | `docs` | `style` | `test` | `chore`

예시:
```
feat(execution): 페이퍼 매수/매도에 슬리피지·수수료 반영

- KRX/US 마찰비용 모델 config 분리
- 단일 트랜잭션으로 현금·포지션 갱신

closed #4
```

**커밋 메시지에 `Co-Authored-By` 또는 Claude/AI 서명·트레일러를 절대 추가하지 않는다.**
